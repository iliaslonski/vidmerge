# vidmerge

**Склейка минутных видео с камер Xiaomi в один файл на Mac.**

📖 **[Открыть пошаговую инструкцию →](https://iliaslonski.github.io/vidmerge/readme.html)**

Bash-скрипт для macOS: собирает минутные `mp4`-клипы от smart-камер Xiaomi (C300, C400, Mi Home Security Camera 360° и аналогичных) в одно длинное видео. Использует `ffmpeg` concat demuxer с флагом `-c copy` — несколько секунд на час видео, без потери качества.

## Что в репозитории

## Быстрый старт

Если вы знакомы с Терминалом:

```bash
brew install ffmpeg                       # один раз
cp vidmerge.sh /path/to/folder/with/clips/
cd /path/to/folder/with/clips/
chmod +x vidmerge.sh                      # один раз для каждой копии
./vidmerge.sh                             # склеить всё в папке
./vidmerge.sh --help                      # все опции
```

Полная инструкция со всеми пояснениями (где что нажимать, что делать на каждом шаге, как переносить файлы с камеры) — в [`readme.html`](https://iliaslonski.github.io/vidmerge/readme.html).

## Совместимость

Любая домашняя smart-камера Xiaomi / Mi / Mijia с приложением Mi Home, пишущая на SD-карту минутными файлами в папку `MIJIA_RECORD_VIDEO/YYYYMMDDHH/`. Скрипт понимает два формата имён файлов:

- `1766763309000.mp4` — Mi Home Export (Unix-таймстемп в миллисекундах).
- `29M53S_1780082993.mp4` — прямое чтение с SD-карты (минута/секунда часа + Unix-таймстемп в секундах).

Файлы из обеих источников можно перемешивать в одной папке — скрипт отсортирует по таймстемпу.

## Принцип работы

1. Сканирует текущую папку (или папку из `--dir`) на распознаваемые имена `.mp4`.
2. Сортирует их хронологически по таймстемпу из имени.
3. (Опционально) фильтрует по диапазону `--from` / `--to` / `--duration`.
4. Проверяет, что у первого, среднего и последнего файла одинаковые кодек, разрешение и fps (иначе stream-copy не сработает).
5. Запускает `ffmpeg -f concat -safe 0 -i list -c copy output.mp4` — копирование потоков без перекодирования.
6. Записывает комментарий (по умолчанию — имя папки) в имя файла и в mp4-метаданные `title` / `comment`.
